Pipeline Flow

How the System Works

From uploading your data to downloading an executive PDF report — four stages powered by AI.

1

Upload & Preprocess

Upload CSV, Excel, JSON, XML, or Parquet files. The system auto-cleans: standardizes column names, drops duplicates, converts types, fills nulls (median for numeric, mode for categorical), and caps outliers using IQR.

2

Multi-Agent Pipeline

Six agents execute in sequence: Retriever → Planner → Stylist → Visualizer → Critic. The Critic can reject a chart and trigger a re-render — a true feedback loop via LangGraph.

3

Triple-View Analysis

With 2 datasets, the system analyzes 3 independent views: Dataset A, Dataset B, and their Intersection — each running its own pipeline with separate results.

4

Export & Act

Generate an executive PDF report with embedded charts, or use the ML-powered recommender to find similar records based on cosine similarity.

System Architecture

LangGraph Orchestration

A directed state graph connects all agents. The Visualizer ↔ Critic edges form a conditional feedback loop that re-renders only when the Critic rejects a chart.
